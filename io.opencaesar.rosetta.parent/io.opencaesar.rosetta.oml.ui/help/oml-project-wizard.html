OML Project Wizard

The OML Project Wizard helps you set up a new OML project.

Catalog

This section configures the catalog.xml file used to map abstract URI prefixes to concrete file paths.

The Base URI field is used to specify an initial prefix URI to add to the catalog file. Directories corresponding to the host name and each path segment will be created under the src/oml folder and mapped to the specified Base URI. In addition to the specified prefix URI, a http:// entry is added to the end, mapped to the vocabulary dependencies directory.

After the project is created additional catalog entries may be added by editing catalog.xml.

Bundle

This section configures an initial project bundle. Additional bundles may be added with the OML Ontology Wizard after the project is created.

The bundle may be either a description bundle or vocabulary bundle, specified with the Bundle Kind field.

The Bundle IRI must be prefixed with the Base URI set in the Catalog section above to allow it to be mapped to a file in the src directory. The final path segment in the Bundle IRI is used as the filename (with a .oml extension) and the ontology prefix. The prefix (and separator, which defaults to # may both be changed by editing the .oml file after the project is created.

Gradle

This section specifies whether or not to create a build.gradle file (by selecting the Configure Gradle checkbox) and configures it for your project with tasks for loading dependencies, performing OWL reasoning, and generating Bikeshed documentation.

The Group ID, Version, and Description fields set the corresponding Gradle project properties. The Bikeshed Publish URL field sets the base publish URL when generating Bikeshed documentation.

The Add Foundation Vocabulary Dependency checkbox specifies whether to include the standard Foundation vocabulary as a Gradle dependency of this project. If enabled, the Foundation vocabulary bundle will be included in the project bundle.

The OWL Reasoning and Bikeshed documentation tasks will be configured to use the project bundle as their root ontologies.
